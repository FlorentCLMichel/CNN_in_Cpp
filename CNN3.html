Introduction ¶

Convolutional Neural Networks (CNNs) are among the most widely-used Deep Learning models for image, speech, and language analysis (although newer models like Transformers and more general Xformers are making inroads into these fields). Their distinguishing feature is the convolutional layer, which make use of the translation equivariance (roughly speaking, the fact that the nature of an object does not depend of where it appears in an image) and locality properties of a problem to significantly reduce the number of parameters compared with dense layers with the same size. These properties make CNNs typically faster to train and more easily generalizable to unseen data. Among their main domains of applications are image analysis tasks like object detection (they are important ingredients of real-time detection models like YOLO, RetinaNet, and SSD) and classification. Efficient implementations of CNNs running on either CPU or GPU are available in standard libraries like TensorFlow and PyTorch, making it straightforward to build, train, and test a network with just a few lines of code.

However, using these libraries may not explain much of how CNNs work beyond the high-level description they provide. At least, that was my experience when starting to use them: thanks to the wealth of tutorials available online, I was soon able to build different CNNs, train them on several datasets, evaluate them, and play with the parameters until I got something which worked well while minimizing the training time; however, I felt I did not really understand what a CNN really was. I was able to use it and understood the main steps—data acquisition, data cleaning, feature engineering, hyperparameter selection through cross-validation, model training, and testing—and got a bird's eye view of the convolution, feedforward, and gradient descent algorithms, but I wanted to know more precisely what actually happened when data goes through the CNN, why these models are so efficient, and the reasons for some of their limitations.

To me, the best way to understand something is to actually build it. So, to understand CNNs, I started to write a simple C++ implementation with handles to access it from Python and experiment with it. The present notebook shows some results of this experiment. While this implementation is, obviously, much less efficient than TensorFlow or PyTorch ones (in part because it does not really optimise the use of different CPU cores and does not use the GPU at all), I hope its simplicity may help others who, like me when I started this project, feel intimidated by the complexity of these libraries' source code but want to get a deeper understanding of CNNs than just using them can provide. If you are in this situation, I strongly encourage you to write your own implementation: writing code is often the best way to learn. Feel free to take inspiration from mine, copy the parts you like (but make sure you really understand them!), improve what you would like to be better, and add your own features. Many standard features like batch normalization, skip connections, and optimizers are not present in the current version of the code; implementing them would be a very good way for you to deepen your understanding! And if you don't have the time to write your own implementation (I know only too well that our time is often limited and one has to choose wisely what to devote it to), I hope the code I have written, which focuses on the core aspects of CNNs without recent improvements which make them both more efficient and more complex, may still help you understand some of the inner workings of CNNs.

In any case, if you have any questions, if something is not clear to you, or if you have spotted a mistake in the C++ code or this notebook, please feel free to reach out!

A few words on the implementation ¶

The code for the neural network can be found in the C++ folder. As mentioned above (and as the name of the folder indicates), it is written in C++, and makes use of the Boost.Python library for Python interoperability.

The network itself is defined in the file CNN3.cpp, using layers defined in Layers.cpp and in their respective .cpp files. It can be divided into three parts:

  • Lines 1 to 32 import the required header and source files and define objects that will be required in the following.
  • Lines 34 to 312 define the CNN3 class.
  • Lines 314 to 325 define the Python interface.

The later is the most relevant for using the CNN from Python. It defines a Python class CNN3 with 8 member functions:

  • An initializer with 8 arguments:
    • the width of input images (int),
    • the height of input images (int),
    • the number of input channels (int),
    • the list of filter sizes for each convolution layer (list of ints),
    • the list of numbers of filters for each convolution layer (list of ints),
    • the list of pooling layer strides (list of ints),
    • the list of numbers of neurons for each dense layer (list of ints),
    • the number of different labels (int).
  • An initializer with no argument (it should be used only if loading the model from a file).
  • A forward function performing a forward pass on an image encoded in a 3D NumPy array and a dropout probability (float), and returning an array of probabilities for the different classes.
  • A backprop function performing a backward pass taking the derivative of a loss function with respect to the output, encoded in a NumPy array, and a learning rate and updating the network weights accordingly.
  • A train function taking an image (encoded as a 3D NumPy array), the correct label (int), a learning rate (float) and a dropout probability (float), performing a forward and a backward passes, and returning the result of the former as a Python list.
  • A save function taking a filename (string) and saving the parameters of the network to the corresponding file.
  • A load function taking a filename (string) and loading the parameters of the network from the corresponding file.
  • A forward_la function performing a forward pass on an image encoded in a NumPy array and a label (int) and returning a Python list containing the loss and accuracy (equal to 1 if the image is correctly classified and 0 otherwise).

This class directly refers to the eponymous C++ class. Its member functions are those of the C++ class, with wrapper functions converting between Python and C++ objects when necessary.

The network is made of a number of convolution blocks followed by hidden fully-connected (also called dense layers; we here use these two terms interchangeably) with ReLU activation function, and by an output dense layer with softmax activation function. Each convolution block is made of a convolution layer with valid padding, a ‘ReLU’ layer, and a MaxPool layer. (These layers will be briefly described in Appendix 1.)

The loss function is the cross-entropy of the output $y$ of the network relative to the probability distribution equal to $1$ for the correct label and $0$ for the other ones. It may be written as: $$ L = - \log(y_l), $$ where $L$ is the loss and $l$ is the correct label.

The file Layers.cpp defines some helpful functions and the different layer classes, whose implementations can be found in their respective source files.

Example of use on the MNIST dataset ¶

In this section I show how to use the CNN on the MNIST handwritten digits dataset, one of the simplest image classification tasks. (The dataset contains 70,000 grayscale images of size 28 by 28 with handwritten digits. The aim of the network will be to learn to reckognise which digit is written on each image.) I choose hyperparameters which demonstrate the features of the implementation while keeping the resulting network relatively simple. As we will see, this simple network is good enough to learn quite efficiently. In the next section, I will show the effect of changing some of the hyperparameters on the accuracy.

The CNN implementation can be found in the module CNN3.so. We also need to load the mnist module, which contains the MNIST dataset, functions from the numpy and os modules we will use later, and the pyplot module to do plots.

save_file_name = 'MNIST_CNN_1.cnn'

The following parameters relate to the training and testing phases:

  • n_train_images (positive integer) is the number of train images (up to 60,000 for MNIST),
  • n_test_images (positive integer) is the number of test images (up to 10,000 for MNIST),
  • n_epochs (positive integer) is the number of epochs,
  • n_print (positive integer) determines the frequency with which the accuracy is printed during training (it is printed every n_print steps),
  • learn_rate (positive float) is the learning rate.
In [3]:
n_train_images = 60000 
n_test_images = 10000
n_epochs = 10
n_print = 4000
learn_rate = 0.01

Next, we define the hyperparameters of the network:

  • img_w (positive integer): width of the input images,
  • img_h (positive integer): height of the input images,
  • num_labels (positive integer): the number of different labels (10 for MNIST),
  • CL_size_filters (list of positive integers): filter sizes for the convolution layers,
  • CL_num_filters (list of positive integers): number of filters for the convolution layers,
  • MP_size (list of positive integers): size of the maxpool layers
  • FC_size (list of positive integers): number of neurons for the fully-connected layers, excluding the output layer,
  • p_dropout: the dropout probability.

The lists CL_size_filters, CL_num_filters, and MP_size must have the same size (equal to the number of convolution layers). The length of the list FC_size gives the number of fully-connected layers excluding the output layer. (The number of neurons in the output layer is equal to num_labels.)

Here we work with a CNN with two convolution layers and one hidden fully-connected layer:

def train_CNN(train_images, train_labels):
    
    nsteps_l = [] # number of steps elapsed 
    loss_l = [] # loss function
    acc_l = [] # accuracy

    print('\n\n— Start of training —')
    for epoch in range(n_epochs):
        nsteps_l.append([])
        loss_l.append([])
        acc_l.append([])
        print('\n— Epoch {:d} of {:d} —\n'.format(epoch+1, n_epochs))

        # shuffle the training data
        permutation = np_permutation(len(train_images))
        train_images = train_images[permutation]
        train_labels = train_labels[permutation]

        # set the loss and number of correctly predicted labels to 0
        loss = 0.
        num_correct = 0

        # for each image in the training set, train the CNN and update the loss 
        # and number of correctly-predicted labels
        for i, (im, label) in enumerate(zip(train_images, train_labels)): 
            results = CNN.train(train_images[i][newaxis,:], int(train_labels[i]), 
                                learn_rate, p_dropout)
            loss += results[0]
            num_correct += results[1]
            # if i is a multiple of n_print, print the loss and accuracy            
            if i % n_print == n_print - 1:
                nsteps_l[-1].append(i)
                loss_l[-1].append(loss / n_print)
                acc_l[-1].append(num_correct / n_print)
                print('Step {:d} — average loss: {:.10f}, accuracy: {:.3f}%'\
                      .format(i+1, loss/n_print, num_correct*100./n_print))
                # reset the loss and number of correctly-preducted labels
                loss = 0.
                num_correct = 0

    # save the CNN
    CNN.save(save_file_name)

    # plot the evolution of the loss function
    for e in range(n_epochs):
        plt.plot(nsteps_l[e], loss_l[e], label='epoch {:d}'.format(e+1))
    plt.xlim(0, n_train_images)
    plt.xlabel(r'$n_{\mathrm{steps}}$')
    plt.ylabel(r'$\mathrm{loss}$')
    plt.grid()
    plt.legend()
    plt.tight_layout()
    plt.show()

    # plot the evolution of the accuracy
    for e in range(n_epochs):
        plt.plot(nsteps_l[e], acc_l[e], label='epoch {:d}'.format(e+1))
    plt.xlim(0, n_train_images)
    plt.xlabel(r'$n_{\mathrm{steps}}$')
    plt.ylabel(r'$\mathrm{accuracy}$')
    plt.grid()
    plt.legend()
    plt.tight_layout()
    plt.show()
    
    # plot the evolution of the loss function - epoch to epoch
    plt.plot(range(n_epochs), [l[-1] for l in loss_l])
    plt.xlim(0, n_epochs-1)
    plt.xlabel(r'$\mathrm{epoch}$')
    plt.ylabel(r'$\mathrm{loss}$')
    plt.grid()
    plt.tight_layout()
    plt.show()

    # plot the evolution of the accuracy  - epoch to epoch
    plt.plot(range(n_epochs), [l[-1] for l in acc_l])
    plt.xlim(0, n_epochs-1)
    plt.xlabel(r'$\mathrm{epoch}$')
    plt.ylabel(r'$\mathrm{accuracy}$')
    plt.grid()
    plt.tight_layout()
    plt.show()
    
if not exists(save_file_name):

    # build the CNN 
    CNN = CNN3.CNN3(img_w, img_h, 1, CL_size_filters, CL_num_filters, MP_size, 
                    FC_size, num_labels)

    print('MNIST CNN initialized') 

    # train the CNN
    train_CNN(train_images, train_labels)

Performance ¶

Choice of hyperparameters¶

Let us now try to change some of the hyperparameters of the model, like the number of layers, neurons, and the size of filters, to see their effects. Proper hyperparameter optimization would require keeping track of the validation accuracy to ensure the models are trained over enough epochs and possibly add batch normalization and a learning rate optimizer, which we do not include here to keep the model as simple as possible. More modestly, our aim is to compare a few sets of parameters and look, on specific examples, at the effects of changing one of them. The results shown below should thus be seen as illustrations on a specific case rather than generalizable facts, and could be a first step to a more precise study.

We first define a function training a model over 20 epochs, the last 10 ones being performed with a learning rate divided by 10, and returning the test accuracy.

In [2]:
def test_params(CNN_params, n_epochs, learn_rate, p_dropout,
               train_images, train_labels, test_images, test_labels):

    # build the CNN 
    CNN = CNN3.CNN3(*CNN_params)

    # train the CNN
    for epoch in range(n_epochs):

        # shuffle the training data
        permutation = np_permutation(len(train_images))
        train_images = train_images[permutation]
        train_labels = train_labels[permutation]

        # train the CNN for each image in the training set
        for i, (im, label) in enumerate(zip(train_images, train_labels)): 
            CNN.train(train_images[i][newaxis,:], int(train_labels[i]), learn_rate, p_dropout)
    
    # divide the learning rate by 10
    learn_rate /= 10
    
    # re-train the CNN
    for epoch in range(n_epochs):

        # shuffle the training data
        permutation = np_permutation(len(train_images))
        train_images = train_images[permutation]
        train_labels = train_labels[permutation]

        # train the CNN for each image in the training set
        for i, (im, label) in enumerate(zip(train_images, train_labels)): 
            CNN.train(train_images[i][newaxis,:], int(train_labels[i]), learn_rate, p_dropout)

    # test the CNN
    num_correct = 0
    for i, (im, label) in enumerate(zip(test_images, test_labels)):
        results = CNN.forward_la(im[newaxis,:], int(label))
        num_correct += results[1]

    return num_correct / len(test_images)

Conclusion ¶

To summarize, we have implemented a simple convolutional neural network in C++ with wrappers for easy access from Python, and tested it on the MNIST dataset of hand-written digits. We found that, with well chosen parameters, this model can reach an accuracy above 99%, and we briefly explored the effects of changing the number of convolution layers, filter sizes, and number of neurons in the hidden dense layer. The accuracy of the model is similar to that of a TensorFlow implementation, but the later is faster to train due to optimizations not present in our simple C++ implementation.

One point to keep in mind is that the MNIST problem is a rather “easy” one for a neural network, and may thus not accurately reflect its performance on more difficult problems. The results shown here thus have an illustrative purpose only. A case-by-case analysis would be required to see which of them holds on harder problems, which arises often in real-world applications of Machine Learning.

Appendix 1: Brief description of each layer ¶

The layers used to build the network are defined in the file C++/Layers.cpp, with implementations in their respective .cpp files. Each layer has a save method saving its weights and parameters to a file and a load method loading them from a file (we use a plaintext format to make saving and loading easier, although a binary format would be more space efficient), as well as forward and backprop methods performing, respectively, the forward and backward propagation.

In the rest of this appendix, indices without an explicit range are understood to describe the set of their possible values.

Convolutional layer¶

The implementation of this layer can be found in ConvLayer.cpp. Each convolution layer has two parameters: the number of filters $n$ and size $s$ of the filters. It takes a 3D array with fixed shape $(c,w,h)$ as input and returns a 3D array with shape $(n,w-s+1,h-s+1)$. Each filter is a 3D array with shape $(c,s,s)$, where $c$ is the number of channels in the input.

Let $X$ denote the input array. The coefficient $(i,j,k)$ of the output array $Y$ is given by: $$ Y_{i,j,k} = \sum_{l = 1}^n \sum_{m = 1}^c \sum_{a = 1}^s \sum_{b = 1}^s F^{(l)}_{m,a,b} X_{m,i+a-1,j+b-1}, $$ where $F^{(l)}$ denotes the $l$th filter.

In the backpropagation stage, the gradient of the loss function is propagated using the formulae $$ \frac{\partial Y_{i,j,k}}{\partial X_{m,p,q}} = \sum_{l = 1}^n F^{(l)}_{m,p+1-i,q+1-j}, $$ where $F^{(l)}_{m,p+1-i,q+1-i}$ is understood to be equal to 0 if one of its last two indices is smaller than $1$ or larger than $s$, and $$ \frac{\partial Y_{i,j,k}}{\partial F^{(l)}_{m,a,b}} = X_{m,i+a-1,j+b-1}. $$

ReLU layer¶

The implementation of this layer can be found in ReLU.cpp and ReLU_v.cpp. It comes in two variants: ReLU and ReLU_v, differing only by their inputs (a 3D array for the former, to be used in a convolution block, or a vector for the latter, to be used after a dense layer).

The forward pass simply applies the ReLU function to each element of the input, i.e., returning its value if it is positive and 0 if it is not. The input is cached during the forward propagation. In the backpropagation, the derivative of the loss function is returned identically if the input was non-negative or replaced by 0 otherwise.

Maxpool layer¶

The implementation of this layer can be found in MaxPool.cpp. It takes a 3D array as input and splits each of its layers into $n$ by $n$ sub-arrays, where $n$ is the size (or stride) of the MaxPool layer. Only the maximum element of each sub-array is retained, thus dividing the number of elements by $n^2$. During the forward pass, the layer also retains the positons of the values which are kept.

During the backpropagation stage, the gradient of the loss function is propagated identically for the values which were kept and set to 0 for those which were not kept.

Fully-connected layer¶

The implementation of this layer can be found in FullCon.cpp. Each fully-connected layer takes a vector as input and outputs another vector; it has a weight matrix $W$ with size $N$ by $I$ and a bias vector $B$ with size $N$, where $N$ is the number of neurons and $I$ the input size. The output $Y$ (of size $N$) corresponding to the input $X$ (of size $I$) is: $Y = W \cdot X + B$, where $\cdot$ denotes matrix-vector multiplication.

The gradient of the loss function can be propagated backward using the formulae: $$ \frac{\partial Y_i}{\partial X_j} = W_{i,j}, $$ $$ \frac{\partial Y_i}{\partial B_j} = \delta_{i,j}, $$ where $\delta_{i,j}$ is the Kronecker symbol equal to $1$ if $i=j$ and $0$ otherwise, and $$ \frac{\partial Y_i}{\partial W_{j,k}} = \delta_{i,j} X_k. $$

Softmax layer¶

The implementation of this layer can be found in SoftMax.cpp. It is simply a fully-connected layer with a softmax activation function, i.e., if $Y$ denotes the (vector) output of the dense layer, the output of the softmax layer is the vector $Z$ with same size whose components are given by: $$ Z_i = \frac{\exp(Y_i)}{\sum_{k} \exp(Y_k)}, $$ where the sum in the denominator is performed over all the components of $Y$.

In the backpropagation stage, we use that $$ \frac{\partial Z_i}{\partial Y_j} = (\delta_{i,j} - Z_j) Z_i, $$ where $\delta_{i,j}$ is the Kronecker symbol equal to $1$ if $i=j$ and $0$ otherwise, that $\frac{\partial L}{\partial Z_i} = 0$ if $i$ is not the correct label, and that $\frac{\partial L}{\partial Z_i} = - \frac{1}{Z_i}$ if $i$ is the correct label (where $L$ is the loss function).
